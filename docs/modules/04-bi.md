# Module 4 — BI Dashboards (Metabase)

## 1. Requirements
The two scope.md dashboards over the gold star, per
`docs/bi/semantic-model-design.md` and the KPI catalog.

## 2. Architecture
**ADR-0005:** Metabase replaces Power BI (PBI Desktop is Windows-only;
development on macOS; student budget). Metabase runs as a compose service
and connects **as `fiq_api`** — BI access is grant-enforced gold-only,
identical to the API's guarantee (verified: only gold tables visible).
The BI design remains tool-portable and authoritative.

## 3. Design rationale
- **KPI parity realized:** every visual is a SQL question whose canonical
  text lives in `docs/bi/queries/` — one versioned truth per KPI
  (testing-strategy mechanism, now operational).
- Dashboard definitions live in Metabase's app volume; the *queries* are
  the versioned artifact (Metabase serialization is enterprise-tier —
  documented trade-off).
- Dashboard 2 ships knowingly partial: value-gap shortlist + drill-through
  arrive with M5's prediction/explanation tables, as the design sequenced.

## 4. Implementation
Compose service (metabase/metabase, app DB on volume, healthy-warehouse
dependency); `make bi-up`; 10 questions across 2 dashboards:
- **Executive Squad Overview:** 4 KPI cards (squad value €17.14bn,
  76 matches, 223 goals, 2.93/match), value-vs-performance quadrant
  (the primary business question as one picture), rankings bar,
  xG over/underperformance diverging bar.
- **Talent Valuation Intelligence:** age-value curve (classic peak-and-
  decay shape, colored by position), top-25 by value, value by position.

## 5. Testing
**KPI parity verified live:** cards show 76 completed / 223 goals — the
exact figures the dbt reconciliation suite guarantees (goal-sum contract).
Gold-only access verified in Metabase's schema browser. Manual visual
checklist per testing strategy §3; dashboard screenshots archived
(docs/assets/, added on release).
